# Editor

> Архитектура редактора LogicCAD.

Version 0.1

---

# 1. Назначение

Editor является единственной подсистемой, через которую пользователь изменяет схему.

Editor никогда:

* не вычисляет сигналы;
* не соединяет игровые провода;
* не создает Runtime;
* не использует API Circuit Network.

Editor изменяет только Document.

---

# 2. Ответственность

Editor отвечает за:

* создание документа;
* изменение документа;
* отображение документа;
* проверку действий пользователя;
* выполнение команд.

Editor не отвечает за компиляцию.

---

# 3. Общая архитектура

```
Player

↓

GUI

↓

Editor Controller

↓

Command System

↓

Document

↓

Render
```

Editor разделяется на несколько независимых подсистем.

---

# 4. Подсистемы

```
Editor

├── Controller
├── Command System
├── Selection
├── Placement
├── Track Tool
├── Component Tool
├── Delete Tool
├── Rotate Tool
├── Render
├── Validation
└── Viewport
```

Каждая подсистема имеет одну ответственность.

---

# 5. Editor Session

Во время работы существует одна активная сессия.

```
EditorSession

├── document
├── player
├── surface
├── selection
├── tool
└── viewport
```

После закрытия редактора сессия уничтожается.

---

# 6. Document

Document загружается при открытии редактора.

```
Open

↓

Load Document

↓

Create Session

↓

Render
```

Document существует независимо от поверхности редактора.

---

# 7. Workspace

Рабочая область имеет фиксированный размер.

```
64 × 64
```

Размер не изменяется.

Все координаты находятся внутри рабочей области.

---

# 8. Grid

Вся работа выполняется по сетке.

Любой объект занимает целочисленные координаты.

Преимущества:

* отсутствие дробных координат;
* простая сериализация;
* простое построение Net Graph;
* повторяемая компиляция.

---

# 9. Camera

Viewport хранит:

* положение камеры;
* масштаб;
* видимую область.

Viewport не влияет на Document.

---

# 10. Инструменты

В MVP доступны пять инструментов.

```
Selection

Component

Track

Delete

Rotate
```

Каждый инструмент работает независимо.

---

# 11. Tool Controller

Активным может быть только один инструмент.

```
Current Tool

↓

Mouse Event

↓

Tool
```

Инструменты не взаимодействуют друг с другом напрямую.

---

# 12. Selection Tool

Позволяет выбрать:

* компонент;
* дорожку.

В MVP поддерживается только одиночное выделение.

Множественное выделение можно добавить позже.

---

# 13. Component Tool

Размещает компонент.

Процесс:

```
Выбор типа

↓

Предпросмотр

↓

Проверка

↓

Создание Component

↓

Render
```

После размещения выполняется Validation.

---

# 14. Rotate Tool

Поворачивает компонент.

```
North

↓

East

↓

South

↓

West
```

После поворота автоматически пересчитываются позиции портов.

---

# 15. Delete Tool

Удаляет:

* компонент;
* дорожку.

После удаления выполняется:

* Validation;
* перестроение Net;
* обновление отображения.

---

# 16. Track Tool

Самая сложная подсистема редактора.

Пользователь рисует дорожки аналогично конвейерам Factorio.

Поддерживаются:

* прямые сегменты;
* углы;
* Т-образные соединения;
* автоматическое объединение;
* автоматическое разделение.

---

# 17. Track Preview

Во время рисования отображается предварительный маршрут.

```
Mouse

↓

Preview

↓

Click

↓

Commit
```

Пользователь всегда видит результат до размещения.

---

# 18. Snap System

Все действия используют привязку.

Привязка возможна к:

* сетке;
* портам;
* существующим дорожкам.

Это исключает появление "висящих" соединений.

---

# 19. Port Interaction

Дорожка никогда не соединяется напрямую с компонентом.

Соединение возможно только через порт.

```
Track

↓

Port

↓

Component
```

Это фундаментальное правило редактора.

---

# 20. Highlight

Редактор умеет подсвечивать сети.

Если выбран порт:

* зелёный — подсвечиваются все связанные зелёные дорожки;
* красный — подсвечиваются все связанные красные дорожки;
* если дорожка принадлежит обеим сетям, отображается двойная подсветка.

Подсветка вычисляется на основе Net Graph.

---

# 21. Validation

После каждой команды запускается локальная проверка.

Проверяются:

* пересечение компонентов;
* соединение с портами;
* выход за границы рабочей области;
* недопустимые конфигурации.

Validation сообщает об ошибках, но не изменяет документ.

---

# 22. Dirty Flags

Редактор не обновляет всё после каждого действия.

Каждая команда помечает изменённые области.

Пример:

```
Rotate Component

↓

Dirty Geometry

↓

Dirty Nets

↓

Dirty Render
```

Это позволит в будущем оптимизировать работу с большими схемами.

---

# 23. Rendering

Редактор отображает:

* сетку;
* компоненты;
* порты;
* дорожки;
* выделение;
* подсветку сетей;
* предварительное размещение.

Render не изменяет Document.

---

# 24. Command System

Все изменения выполняются через команды.

```
PlaceComponent

DeleteComponent

RotateComponent

PlaceTrack

DeleteTrack
```

Команда изменяет Document и сообщает системе, какие данные стали "грязными".

---

# 25. Ограничения MVP

В первую версию не входят:

* Undo/Redo;
* копирование;
* вставка;
* группировка;
* автотрассировка;
* множественное выделение;
* пользовательские инструменты.

Архитектура должна позволять добавить эти функции без переработки существующих подсистем.
