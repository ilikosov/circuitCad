# Graph Builder

> Построение внутреннего графа схемы.

Version 0.1

---

# 1. Назначение

Graph Builder преобразует пользовательскую геометрию (`Document`) во внутреннее представление (`Graph`), удобное для анализа.

Graph является производной структурой данных.

Он **не хранится** в документе и полностью перестраивается после изменений геометрии.

---

# 2. Почему нужен Graph

Пользователь работает с дорожками.

Компилятор работает со связями.

Graph Builder связывает эти два уровня.

```text
Document
│
├── Components
├── Tracks
└── Metadata

        │

        ▼

Graph Builder

        ▼

Graph
```

---

# 3. Входные данные

Graph Builder использует только:

* Components;
* Tracks;
* Component Definitions.

Он не использует Runtime и API Factorio.

---

# 4. Выходные данные

Результатом является объект `Graph`.

```text
Graph
├── Nodes
├── Edges
└── Metadata
```

---

# 5. Node

Node представляет электрическую точку.

Каждый Node имеет:

* уникальный идентификатор;
* координаты;
* тип;
* список смежных Edge.

---

# 6. Типы Node

Поддерживаются следующие типы:

```text
Port
Junction
Endpoint
```

### Port

Порт компонента.

### Junction

Точка пересечения дорожек.

### Endpoint

Свободный конец дорожки.

---

# 7. Edge

Edge соединяет два Node.

```text
Node ---- Edge ---- Node
```

Edge:

* не имеет направления;
* принадлежит одной цветовой сети (Red или Green);
* имеет длину (для геометрического анализа).

---

# 8. Этапы построения

Graph Builder выполняет следующие шаги:

1. Создание Node для всех портов компонентов.
2. Создание Node для концов дорожек.
3. Создание Junction в местах соединения дорожек.
4. Создание Edge между Node.
5. Объединение совпадающих Node.
6. Проверка целостности графа.

---

# 9. Построение Port Node

Для каждого компонента:

1. Получить `Component Definition`.
2. Вычислить положение портов с учётом поворота.
3. Создать Node типа `Port`.

---

# 10. Построение Track Node

Каждая дорожка имеет:

* начало;
* конец.

Для каждой точки создаётся Node.

Если в этой координате уже существует Node, используется существующий.

---

# 11. Построение Junction

Если несколько дорожек сходятся в одной координате, создаётся один общий Node типа `Junction`.

Все Edge подключаются к нему.

---

# 12. Объединение Node

После построения все Node с одинаковыми координатами объединяются.

Это позволяет автоматически формировать Т-образные соединения.

---

# 13. Построение Edge

Каждый сегмент дорожки создаёт один Edge.

Edge содержит:

* начало;
* конец;
* цвет (Red/Green).

---

# 14. Проверка графа

После построения выполняется проверка:

* нет ли Edge без Node;
* нет ли Node без координат;
* существуют ли все ссылки;
* нет ли циклических ссылок в структуре данных.

---

# 15. Инварианты

После завершения Graph Builder гарантируется:

* каждый Edge соединяет ровно два Node;
* каждый Node имеет корректный тип;
* отсутствуют дублирующиеся Node в одной координате;
* граф согласован.

---

# 16. Производительность

Для поиска Node по координатам рекомендуется использовать пространственный индекс.

Например:

```text
Map<(x, y), NodeId>
```

Это обеспечивает построение графа за время, близкое к линейному относительно числа дорожек.

---

# 17. Обновление графа

В MVP граф полностью перестраивается после каждого изменения геометрии.

Инкрементальное обновление откладывается на будущие версии.

---

# 18. Использование Graph

Graph используется следующими подсистемами:

* Validation;
* Highlight;
* Net Builder;
* Compiler.

Document продолжает оставаться единственным источником истины.

---

# 19. Ошибки

Graph Builder может сообщать о следующих ошибках:

* недопустимая геометрия;
* пересечение вне сетки;
* отсутствующий порт;
* повреждённая структура документа.

Ошибки не исправляются автоматически.
