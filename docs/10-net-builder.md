# Net Builder

> Построение логических сетей (Net) на основе Graph.

Version 0.1

---

# 1. Назначение

Net Builder преобразует граф соединений (`Graph`) в набор независимых логических сетей (`Net`).

Каждая сеть соответствует одной электрической цепи одного цвета.

Graph описывает геометрию.

Net описывает электрические соединения.

---

# 2. Входные данные

Net Builder получает:

* Graph;
* Component Definitions.

Он не использует Document напрямую.

---

# 3. Выходные данные

Результатом является объект `NetGraph`.

```text
NetGraph
├── Nets
└── Statistics
```

`NetGraph` используется компилятором.

---

# 4. Net

Net представляет одну непрерывную электрическую сеть.

```text
Net
├── id
├── color
├── nodes
├── ports
└── metadata
```

Каждая сеть имеет уникальный идентификатор.

---

# 5. Цвет сети

В MVP существует два цвета.

```text
Red
Green
```

Сети разных цветов никогда не объединяются, даже если геометрически проходят по одним и тем же координатам.

---

# 6. Основной алгоритм

Net Builder выполняет поиск связных компонент графа.

Для каждого ещё не посещённого узла:

1. Запустить обход графа (DFS или BFS).
2. Посетить все достижимые узлы того же цвета.
3. Создать новый `Net`.
4. Присвоить всем найденным узлам идентификатор сети.

---

# 7. Псевдокод

```text
for each node:
    if node.visited:
        continue

    net = new Net()

    flood_fill(node)

    add(net)
```

Алгоритм линейный относительно числа узлов и рёбер.

---

# 8. Обход

Во время обхода:

* помечаются посещённые узлы;
* собираются все порты;
* подсчитывается статистика сети.

---

# 9. Порты

Каждый порт принадлежит одной или нескольким сетям.

Для MVP:

* красный канал → Red Net;
* зелёный канал → Green Net.

Если порт не подключён, соответствующая сеть отсутствует.

---

# 10. Изолированные сети

Даже сеть без компонентов считается корректной.

Это позволяет пользователю строить схему постепенно.

Такие сети могут подсвечиваться как предупреждение.

---

# 11. Висячие окончания

Если сеть заканчивается Endpoint, это не является ошибкой.

Однако Validation может выдать предупреждение.

---

# 12. Пересечения

Если дорожки пересекаются и соединяются, они входят в одну сеть.

Если в будущих версиях появятся мостики или многослойность, Graph Builder создаст разные узлы, и Net Builder автоматически сформирует независимые сети.

---

# 13. Структура Net

Каждая сеть хранит:

* идентификатор;
* цвет;
* список узлов;
* список портов;
* количество соединений;
* ограничивающий прямоугольник (Bounding Box).

Bounding Box используется для ускорения подсветки и анализа.

---

# 14. Использование

`NetGraph` используется:

* подсветкой;
* Validation;
* Compiler;
* инструментами анализа.

Ни одна подсистема не работает напрямую с Graph после построения сетей.

---

# 15. Validation

Net Builder может обнаружить:

* пустые сети;
* сети без портов;
* слишком большие сети;
* повреждённые графы.

Ошибки возвращаются вызывающему коду.

---

# 16. Производительность

Алгоритм должен работать за время O(V + E), где:

* V — количество узлов;
* E — количество рёбер.

Это гарантирует хорошую масштабируемость даже для крупных схем.

---

# 17. Расширяемость

В будущем Net Builder должен поддерживать:

* многослойные дорожки;
* разные типы проводников;
* специальные типы соединений;
* пользовательские правила объединения.

Архитектура алгоритма не должна изменяться.

---

# 18. Инварианты

После выполнения Net Builder гарантируется:

* каждый узел принадлежит не более чем одной сети одного цвета;
* каждая сеть является связной;
* отсутствуют дублирующиеся сети;
* все порты корректно связаны с сетями.

---

# 19. Результат

После построения `NetGraph` геометрическая информация больше не требуется компилятору.

Компилятор работает только с:

* компонентами;
* портами;
* сетями.
