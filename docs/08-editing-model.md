# Editing Model

> Модель редактирования схем LogicCAD.

Version 0.1

---

# 1. Назначение

Документ описывает, как действия пользователя изменяют Document.

Editor не изменяет данные напрямую.

Все изменения выполняются через команды.

---

# 2. Основной принцип

Редактор является транзакционной системой.

Любое действие пользователя — это атомарная операция.

```text
Mouse Event
      │
      ▼
Editor Command
      │
      ▼
Validation
      │
      ▼
Commit
      │
      ▼
Render
```

Если команда завершается ошибкой, Document остается неизменным.

---

# 3. Command Pipeline

Каждая команда проходит одинаковый жизненный цикл.

```text
Create

↓

Preview

↓

Validate

↓

Execute

↓

Commit

↓

Render
```

---

# 4. Типы команд

В MVP существуют следующие команды.

```text
Create Component

Delete Component

Rotate Component

Move Component

Create Track

Delete Track
```

Позже появятся:

* Copy
* Paste
* Group
* Undo
* Redo

---

# 5. Preview

Во время предварительного размещения изменения не попадают в Document.

Создается временная модель.

```text
Document

+

Preview Layer
```

Preview Layer уничтожается после завершения команды.

---

# 6. Commit

После успешной проверки изменения применяются к Document.

Команда должна быть неделимой.

Нельзя сохранить только часть изменений.

---

# 7. Отмена

Если Validation обнаруживает ошибку,

команда отменяется полностью.

Документ остается в прежнем состоянии.

---

# 8. Dirty Flags

После выполнения команды определяются измененные подсистемы.

Например:

```text
Move Component

↓

Geometry Dirty

↓

Graph Dirty

↓

Render Dirty
```

Каждая подсистема обновляется независимо.

---

# 9. Geometry Update

Изменение геометрии включает:

* добавление дорожек;
* удаление дорожек;
* перемещение компонентов;
* поворот компонентов.

После изменения Geometry Graph Builder должен перестроить граф.

---

# 10. Graph Update

Graph Builder никогда не вызывается напрямую инструментами.

Инструменты только помечают граф как устаревший.

```text
Tool

↓

Dirty Graph

↓

Graph Builder
```

Это позволяет позже реализовать ленивое обновление.

---

# 11. Validation

Validation работает на двух уровнях.

## Быстрая проверка

Выполняется во время Preview.

Например:

* выход за границы;
* пересечение компонентов.

## Полная проверка

Выполняется после Commit.

Например:

* построение графа;
* проверка связности;
* проверка допустимости конфигурации.

---

# 12. Инструменты

Каждый инструмент отвечает только за интерпретацию действий пользователя.

Он не знает внутреннего устройства документа.

```text
Mouse

↓

Tool

↓

Command
```

---

# 13. Component Tool

Алгоритм:

```text
Выбор типа

↓

Preview

↓

Validation

↓

Create Component

↓

Commit
```

---

# 14. Track Tool

Алгоритм:

```text
Начало трассы

↓

Добавление сегментов

↓

Предпросмотр

↓

Подтверждение

↓

Commit
```

Track Tool не строит Net.

Он изменяет только геометрию.

---

# 15. Rotate Tool

При повороте:

* изменяется Rotation;
* пересчитываются координаты портов;
* граф помечается как устаревший.

---

# 16. Delete Tool

Удаление может затронуть:

* компонент;
* дорожку;
* несколько дорожек.

После удаления запускается перестроение графа.

---

# 17. Move Tool

Компонент можно перемещать только целиком.

Во время перемещения:

* отображается Preview;
* дорожки не перестраиваются автоматически;
* окончательная проверка выполняется после подтверждения.

---

# 18. Render Synchronization

Render не читает команды.

Он отображает только текущее состояние Document и Preview Layer.

---

# 19. Transaction Model

Несколько изменений могут быть объединены в одну транзакцию.

Например:

```text
Move Component

+

Reconnect Track

↓

One Commit
```

Это станет основой для будущего Undo/Redo.

---

# 20. Event Sequence

Полная последовательность выглядит следующим образом.

```text
Input

↓

Tool

↓

Command

↓

Preview

↓

Validation

↓

Commit

↓

Dirty Flags

↓

Graph Builder

↓

Render
```

---

# 21. Расширяемость

Новая функциональность должна реализовываться через новые команды.

Существующие инструменты не должны изменяться.
