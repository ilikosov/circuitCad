# LogicCAD for Factorio

> CAD-редактор логических схем для Factorio с использованием встроенного движка Circuit Network.

Version: 0.1 (Draft)

---

# 1. Введение

LogicCAD — это мод для Factorio, предоставляющий специализированный редактор логических схем.

Вместо построения схем непосредственно на игровой карте пользователь работает в отдельном редакторе, напоминающем классические CAD-системы для цифровой электроники.

После завершения редактирования схема автоматически компилируется в реальные комбинаторы Factorio и выполняется штатным движком Circuit Network.

Таким образом LogicCAD не является собственной системой симуляции.

Он представляет собой графический редактор и компилятор.

Все вычисления выполняются исключительно игровым движком.

---

# 2. Цели проекта

Проект преследует несколько целей.

## Упростить разработку больших схем

Обычные схемы Factorio быстро становятся трудно читаемыми.

Большое количество проводов затрудняет навигацию.

LogicCAD заменяет провода дорожками, похожими на дорожки печатных плат.

---

## Повысить читаемость

В редакторе используются:

* явные порты
* дорожки
* автоматическая подсветка сетей
* регулярная сетка
* единый стиль компонентов

В результате большие проекты становятся значительно проще для понимания.

---

## Использовать существующий движок Factorio

LogicCAD намеренно не реализует собственную симуляцию логики.

Причины:

* полная совместимость с игрой
* отсутствие различий между редактором и игровым процессом
* отсутствие необходимости поддерживать собственный логический движок

После компиляции схема становится обычным набором комбинаторов.

---

# 3. Основная идея

Архитектура проекта выглядит следующим образом.

```text
               +----------------------+
               |     LogicCAD         |
               +----------+-----------+
                          |
              +-----------+------------+
              |                        |
         Editor Surface          Runtime Surface
              |                        |
         Components             Arithmetic
         Tracks                 Decider
         Ports                  Constant
              |                        |
              +-----------+------------+
                          |
                    Circuit Network
```

Редактор является единственным источником истины.

Runtime является полностью сгенерированной копией.

---

# 4. Основные принципы

Проект строится вокруг нескольких архитектурных принципов.

## Editor First

Редактор является источником истины.

Runtime никогда не изменяется вручную.

Все изменения выполняются исключительно через редактор.

---

## Runtime Disposable

Runtime можно полностью удалить.

После этого он полностью восстанавливается из редактора.

Это гарантирует отсутствие рассинхронизации.

---

## WYSIWYG

Компилятор не изменяет пользовательскую схему.

Положение компонентов сохраняется.

Топология соединений сохраняется.

Ориентация компонентов сохраняется.

---

## No Logic Simulation

LogicCAD никогда не вычисляет сигналы самостоятельно.

Все вычисления выполняет движок Factorio.

---

## CAD Instead of Sandbox

LogicCAD является CAD-приложением.

Это означает использование:

* компонентов
* портов
* дорожек
* сетей
* компилятора

вместо свободного размещения проводов.

---

# 5. Что входит в MVP

Первая версия проекта включает только минимально необходимый функционал.

## Компоненты

* Arithmetic Combinator
* Decider Combinator
* Constant Combinator

---

## Редактор

* отдельная поверхность
* размер 64×64
* один активный редактор
* рисование дорожек
* размещение компонентов
* поворот компонентов
* удаление компонентов

---

## Компилятор

Компилятор выполняет:

1. проверку схемы
2. построение Net Graph
3. удаление Runtime
4. создание Runtime
5. соединение проводами

---

## Runtime

Runtime располагается на скрытой поверхности.

Пользователь не взаимодействует с ним напрямую.

---

# 6. Что не входит в MVP

Следующие возможности сознательно отложены.

* Undo / Redo
* пользовательские компоненты
* библиотека компонентов
* автотрассировка
* автоматическое размещение
* оптимизирующий компилятор
* многослойные дорожки
* многопользовательское редактирование
* собственная панель свойств

Архитектура проекта учитывает возможность их будущего добавления.

---

# 7. Терминология

| Термин | Значение |
|---------|----------|
| Document | Полная схема редактора |
| Component | Логический блок редактора |
| Port | Точка подключения дорожек |
| Track | Дорожка редактора |
| Net | Логическая сеть |
| Compiler | Преобразователь Document → Runtime |
| Runtime | Сгенерированная схема Factorio |
| Runtime Builder | Подсистема создания сущностей Factorio |
