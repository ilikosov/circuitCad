# Track System

> Архитектура дорожек LogicCAD.

Version 0.1

---

# 1. Назначение

Track System отвечает за построение геометрии электрических соединений.

Его задача — предоставить пользователю удобный способ соединения компонентов.

Track System не отвечает за выполнение логики.

---

# 2. Главный принцип

Дорожки являются только геометрией.

Электрическая сеть вычисляется отдельно.

```text
Geometry

↓

Graph

↓

Net
```

---

# 3. Представление

Дорожка состоит не из объекта Track.

Она представляет собой множество рёбер графа.

```text
Vertex

↓

Edge

↓

Vertex
```

---

# 4. Vertex

Вершина является основной единицей системы.

Каждая вершина имеет:

* идентификатор;
* координату;
* тип.

---

# 5. Типы Vertex

Поддерживаются четыре типа.

### Port

Порт компонента.

---

### Junction

Точка соединения дорожек.

---

### Endpoint

Конец дорожки.

---

### Internal

Служебная вершина.

Используется длинными дорожками.

---

# 6. Edge

Edge соединяет две вершины.

```text
Vertex

↓

Edge

↓

Vertex
```

Edge не имеет направления.

---

# 7. Координаты

Все вершины располагаются на сетке.

Используются только целые координаты.

---

# 8. Построение

Во время рисования дорожки редактор выполняет:

1. создание вершин;
2. создание рёбер;
3. объединение совпадающих вершин;
4. обновление графа.

---

# 9. Объединение

Если две вершины занимают одинаковую координату,

они автоматически объединяются.

Это позволяет реализовать T-образные соединения.

---

# 10. Разделение

После удаления дорожки выполняется анализ графа.

Если граф разделился,

создаются новые вершины Endpoint.

---

# 11. Пересечение

В MVP пересечение автоматически означает соединение.

Позже можно добавить "мостики" или многослойность.

---

# 12. Привязка

Во время рисования вершины автоматически привязываются:

* к портам;
* к существующим вершинам;
* к сетке.

---

# 13. Подсветка

Подсветка выполняется обходом графа.

Никаких специальных структур хранения не требуется.

---

# 14. Validation

Проверяются:

* дублирующиеся вершины;
* недопустимые рёбра;
* изолированные вершины;
* висячие окончания.

---

# 15. Удаление

Удаление Edge может привести к разделению графа.

После удаления выполняется перестроение связных компонент.

---

# 16. Runtime

Компилятор никогда не использует геометрию напрямую.

Он получает уже готовый Net Graph.

---

# 17. Будущие возможности

Архитектура позволяет добавить:

* многослойные дорожки;
* мостики;
* автоматическую трассировку;
* разные типы проводников;
* оптимизацию маршрутов.

Без изменения структуры графа.
