# Domain Model

> Формальная модель данных LogicCAD.

Version: 0.1 (Draft)

---

# 1. Назначение

Данный документ описывает все основные сущности проекта.

Любая подсистема LogicCAD работает исключительно с моделями, определенными в этом документе.

Это означает:

* Editor изменяет модели;
* Compiler читает модели;
* Runtime Builder использует результат компиляции;
* GUI отображает модели.

Ни одна из моделей не зависит от API Factorio.

---

# 2. Основной принцип

LogicCAD разделяется на два уровня.

```
Editor Model

↓

Compiler Model

↓

Runtime
```

Editor Model является источником истины.

Runtime никогда не хранит пользовательские данные.

---

# 3. Главная сущность — Document

Вся схема представляется одним объектом.

```
Document
```

Document является аналогом файла проекта.

Он содержит абсолютно всю информацию, необходимую для восстановления схемы.

---

# 4. Document

```
Document
│
├── Metadata
├── Components
├── Tracks
├── Viewport
└── Version
```

---

## 4.1 Metadata

Метаданные схемы.

Пример:

```text
Name

Description

Author

Created

Modified
```

---

## 4.2 Components

Список всех компонентов редактора.

Каждый компонент имеет уникальный идентификатор.

```
ComponentId
```

Идентификаторы никогда не переиспользуются внутри документа.

---

## 4.3 Tracks

Все дорожки редактора.

Хранятся независимо от компонентов.

---

## 4.4 Viewport

Служебная информация редактора.

Например:

* положение камеры;
* масштаб;
* выделение.

Эти данные не участвуют в компиляции.

---

## 4.5 Version

Версия формата документа.

Используется исключительно для миграций.

---

# 5. Component

Компонент является логическим блоком редактора.

Он не связан с сущностями Factorio.

```
Component
│
├── id
├── type
├── position
├── rotation
├── ports
└── configuration
```

---

## id

Уникальный идентификатор.

Используется всеми остальными моделями.

---

## type

Тип компонента.

Например

```
Arithmetic

Decider

Constant
```

В дальнейшем список расширяется.

---

## position

Положение на сетке.

```
x

y
```

Используются только целочисленные координаты.

---

## rotation

Направление.

```
North

East

South

West
```

---

## ports

Описание портов.

Компонент никогда не вычисляет их автоматически.

Порты принадлежат компоненту.

---

## configuration

Все пользовательские настройки.

Например

Arithmetic

```
Operation

Left Signal

Right Signal

Output Signal
```

---

# 6. Port

Port является точкой подключения дорожек.

```
Port
│
├── id
├── owner
├── side
├── color
└── position
```

---

## owner

Идентификатор компонента.

---

## side

```
Left

Right

Top

Bottom
```

После поворота компонента расположение пересчитывается.

---

## color

```
Green

Red
```

---

## position

Локальная координата относительно компонента.

---

# 7. Track

Track представляет собой сегмент дорожки.

```
Track
│
├── id
├── start
├── end
└── layer
```

---

## start

Координата начала.

---

## end

Координата конца.

---

## layer

Для MVP всегда

```
0
```

Позже позволит реализовать многослойные дорожки.

---

# 8. Junction

Точка соединения нескольких дорожек.

```
Junction
```

Создается автоматически.

Пользователь отдельно Junction не размещает.

---

# 9. Net

Net является результатом анализа дорожек.

```
Net
│
├── id
├── tracks
├── ports
└── color
```

Net никогда не хранится в документе.

Он строится заново.

---

# 10. Runtime Package

Результат работы компилятора.

```
RuntimePackage
│
├── Entities
├── Wires
└── Metadata
```

Runtime Package существует только во время компиляции.

Он не сериализуется.

---

# 11. Связи между моделями

```
Document

├── Components

│     └── Ports

│

└── Tracks

↓

Net Builder

↓

Net Graph

↓

Compiler

↓

Runtime Package
```

---

# 12. Что сериализуется

Сохраняются:

* Document
* Components
* Tracks
* Metadata
* Version

---

Не сохраняются:

* Net
* Runtime Package
* Runtime
* Graph

---

# 13. Требования

Все модели должны:

* быть независимыми от API Factorio;
* поддерживать сериализацию;
* поддерживать копирование;
* поддерживать сравнение;
* иметь стабильную структуру.

---

# 14. Запрещено

Модели не должны:

* хранить LuaEntity;
* хранить LuaSurface;
* хранить LuaPlayer;
* хранить ссылки на объекты Factorio.

Все подобные ссылки принадлежат исключительно Runtime.

---

# 15. Жизненный цикл

```
Создание документа

↓

Редактирование

↓

Валидация

↓

Построение Net

↓

Компиляция

↓

Runtime Package

↓

Создание Runtime

↓

Работа схемы
```

---

# 16. Будущее развитие

Модель данных должна без изменений поддерживать:

* пользовательские компоненты;
* библиотеку компонентов;
* вложенные компоненты;
* шаблоны;
* параметризованные компоненты;
* импорт и экспорт.
